C-06
Fix Booking Page Mobile Performance (45 → 75+ target)
High Effort
What to do
  1. Install WP Rocket (paid, recommended) or LiteSpeed Cache (free if on LiteSpeed server). Enable page caching, browser caching, and GZIP compression.
  2. In WP Rocket or your caching plugin, enable lazy loading for all images and enable minification for CSS and JS files.
  3. Run the site through TinyPNG.com or install Imagify / ShortPixel WordPress plugin to compress all existing images. Target: under 200KB per image.
  4. In your caching plugin, use the Defer JS option to delay non-critical scripts from loading on the booking page.
  5. Go to your hosting control panel — if on shared hosting, consider upgrading to a VPS or managed WordPress host (WP Engine, Kinsta, or Flywheel). Current server response time is 1,007ms; target is under 200ms.
  6. After each change, re-run PageSpeed Insights on the booking page URL to confirm the mobile score is improving. Target: 75+.
