2. Key Decisions

2.1 How we run it in Smartlead Recommended

Add BlankSlate as a Client in our existing Smartlead account (Client Access → Add Client). No separate account needed. Every campaign gets assigned to the BlankSlate client in campaign General Settings, which keeps their data, analytics, and Master Inbox view cleanly separated from other work.

  • Our skill already supports this: campaigns.py create --client-id <id>
  • Optional: white-label client workspace is a $29/mo add-on per client — gives Derek a branded login that shows only his campaigns. Only worth it if we want him self-serve in the platform; otherwise our own reporting covers it. Decide alongside the reporting decision (2.4).

2.2 Sending infrastructure — branded alternate domains Kick off now

Derek wants branded alternate domains that BlankSlate owns (not Smartlead-provisioned mailboxes). This is the longest lead-time item because of warmup, so it starts first.

The math on volume: 7,000 sends/mo ≈ 230–320/day at full ramp. Safe cold volume is ~25–30/day per mailbox, and best practice is 2–3 mailboxes per domain. That gives us:

Full step-by-step in Section 3: Infrastructure Setup Runbook below.

Timeline lever: aged domains (ones BlankSlate already owns) can go live after 2–3 weeks of warmup; brand-new registrations need ~30 days — Microsoft now weighs domain age + sending consistency heavily. Ask Derek if he already owns candidate domains; it buys back two weeks.

2.3 Multi-channel (LinkedIn etc.) Phase 2 — not launch-blocking

Confirmed: Smartlead's LinkedIn steps work through integration partners (HeyReach or Aimfox), not fully native automation. Smartlead does natively support subsequences with conditional logic (e.g., opened-but-no-reply branches). Plan: launch email-only, prove reply rates, then propose LinkedIn layering via HeyReach as an optimization — potentially tied to the Phase 2 Founder Brand add-on already in the deck.

2.4 Reporting Decide at training call

Two options, decide together at the handoff call:

  • Option A: Smartlead white-label portal ($29/mo) — Derek logs in and sees live campaign stats himself
  • Option B: Skill-generated monthly report — we pull send/open/reply/positive-reply metrics via the API and generate a branded report URL, folded into the monthly reporting across all initiatives

Option B fits the existing monthly reporting deliverable and costs nothing; A adds client self-serve transparency. They're not mutually exclusive.

3. Infrastructure Setup Runbook

End-to-end process for domains → mailboxes → Smartlead → warmup. Active work is only a few hours total; calendar time is almost all warmup, which is why this starts before anything else.

Phase 1 — Get the domains Client / Melissa

In order of preference:

  1. Domains BlankSlate already owns — aged domains warm faster and are trusted sooner
  2. They buy 3–4 new ones — variants like blankslatehq.com, tryblankslate.com, getblankslate.com
  3. DNS access — delegated registrar/DNS access, or a client contact who pastes records same-day. Slow DNS turnaround is the classic bottleneck.

Phase 2 — Mailboxes on Google Workspace ~1 hour

Two ways to structure it — decide with client:

  • Cleanest: add the alt domains as secondary domains in BlankSlate's existing Google Workspace, create users on them. They pay the seats (~$7/user/mo), they own everything, nothing to unwind at offboarding.
  • Alternative: a fresh Workspace tenant just for outbound that we admin. More control for us, but someone owns the billing conversation.

Create 2–3 users per domain using real sender personas (e.g., derek@, d.lastname@) — 9–12 mailboxes total. Set display name, photo, and signature on each so they look human.

Phase 3 — DNS records per domain ~30 min

RecordWhat
MXGoogle Workspace MX records
SPFv=spf1 include:_spf.google.com ~all
DKIMGenerate in Google Admin → paste TXT record
DMARCv=DMARC1; p=none; rua=mailto:... — monitor mode, don't enforce yet
Tracking CNAMECustom tracking domain per sending domain (e.g., track.blankslatehq.com) — never Smartlead's shared one
Forwarding301 redirect the bare domain → main BlankSlate site, so anyone checking the domain lands somewhere real

Verify SPF/DKIM/DMARC all pass before connecting anything: send a test to a Gmail account and check "Show original."

Phase 4 — Connect to Smartlead ~30 min

Use Google OAuth, not app passwords — OAuth survives Google security-policy changes that silently disconnect IMAP/app-password mailboxes mid-campaign.

  1. In Smartlead: Email Accounts → Connect Mailbox → Google OAuth — it shows Smartlead's Client ID
  2. In Google Admin Console → Security → API Controls → Manage App Access → Configure New App — paste that Client ID and trust the app. One-time step per Workspace (~10 min to propagate); unlocks one-click OAuth for every mailbox on the tenant
  3. Back in Smartlead, connect each of the 9–12 mailboxes via the OAuth pop-up and assign them to the BlankSlate client workspace

Phase 5 — Warmup settings 5 min per mailbox, then wait

Enable warmup on every mailbox the day it connects:

  • Total warmup emails/day: cap around 40, starting low
  • Ramp-up increment: +4/day, or use Smartlead's auto-adjust (throttles the ramp based on each inbox's health score)
  • Reply rate: 50–60% to simulate real conversations
  • Randomize daily volume so the pattern looks human
  • Warmup never turns off — it runs alongside live campaigns permanently; teams that kill it post-launch see deliverability decay within 6–8 weeks
Gate before first cold send: 2–3 weeks of warmup minimum (30 days for brand-new domains), health scores green on every mailbox, and a spam-placement test showing inbox — not promotions/spam — on both Gmail and Outlook. When campaigns start, per-mailbox cold volume begins ~10/day and ramps toward 25–30/day, which is exactly what the SOW's 10-week ramp absorbs.
