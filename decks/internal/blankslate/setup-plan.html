2. Key Decisions

2.1 How we run it in Smartlead Recommended

Add BlankSlate as a Client in our existing Smartlead account (Client Access → Add Client). No separate account needed. Every campaign gets assigned to the BlankSlate client in campaign General Settings, which keeps their data, analytics, and Master Inbox view cleanly separated from other work.

  • Our skill already supports this: campaigns.py create --client-id <id>
  • Optional: white-label client workspace is a $29/mo add-on per client — gives Derek a branded login that shows only his campaigns. Only worth it if we want him self-serve in the platform; otherwise our own reporting covers it. Decide alongside the reporting decision (2.4).

2.2 Sending infrastructure — branded alternate domains Kick off now

Derek wants branded alternate domains that BlankSlate owns (not Smartlead-provisioned mailboxes). This is the longest lead-time item because of warmup, so it starts first.

The math on volume: 7,000 sends/mo ≈ 230–320/day at full ramp. Safe cold volume is ~25–30/day per mailbox, and best practice is 2–3 mailboxes per domain. That gives us:

Full step-by-step in Section 3: Infrastructure Setup Runbook below.

Timeline lever: aged domains (ones BlankSlate already owns) can go live after 2–3 weeks of warmup; brand-new registrations need ~30 days — Microsoft now weighs domain age + sending consistency heavily.

3. Infrastructure Setup Runbook

End-to-end process for domains → mailboxes → Smartlead → warmup. Active work is only a few hours total; calendar time is almost all warmup, which is why this starts before anything else.

Phase 0 — The client ask Sent Jul 9

One-shot access request sent to the client as a Google Doc (ET - Sales & CS → Blank Slate → "BlankSlate Outbound Email — One-Time Setup"). It bundles everything so we only ask once:

  • Confirmations: renew blankslatetechnologies.com (expires Jul 31), confirm blankslate.app mail is dormant, buy 2 domain variants
  • Option A (recommended): they share DNS access (Iwantmydomain.com) + a temporary Super Admin user on their Workspace + billing heads-up for ~$7/user/mo seats. We do everything, they revoke admin after.
  • Option B: they do the Google Admin steps themselves (add secondary domains, create users, whitelist the Smartlead client ID) using the doc's click-paths, then send us mailbox logins via password manager; we send them the DNS records.

Note: the Smartlead white-label client portal does NOT help with setup — client permissions there only cover viewing/managing campaigns, assigned email accounts, and leads. The setup bottleneck is Google Admin + DNS, and mailbox connections live in our Smartlead account where the campaigns run. Portal is a reporting decision for later.

Phase 1 — Get the domains Client / Melissa

Resolved Jul 9 — see domain diligence table in §2.2. Sending set: blankslateapp.com + blankslate.app (pending mail-use confirmation) + 2 new purchases. DNS lives at Iwantmydomain.com (existing) and wherever the new ones are bought.

Phase 2 — Mailboxes on Google Workspace ~1 hour

Confirmed Jul 9: BlankSlate runs Google Workspace on blankslatetech.co. Route: add the sending domains as secondary domains in their existing Workspace (Google Admin → Account → Domains → Manage domains → Add a domain → secondary domain, verify via TXT). They pay the seats (~$7/user/mo), they own everything, nothing to unwind at offboarding.

Create 2–3 users per domain as Derek variations (e.g., derek@, d.lastname@, derekw@) — 9–12 mailboxes total across the 4 domains. Set display name, photo, and signature on each so they look human.

Verify SPF/DKIM/DMARC all pass before connecting anything: send a test to a Gmail account and check "Show original."

Phase 4 — Connect to Smartlead ~30 min

Use Google OAuth, not app passwords — OAuth survives Google security-policy changes that silently disconnect IMAP/app-password mailboxes mid-campaign.

  1. In Smartlead: Email Accounts → Add → Google OAuth — copy Smartlead's Client ID from the pop-up
  2. In Google Admin Console → Security → API Controls → Manage App Access → Configure New App — paste the Client ID, select Smartlead, scope "All users" (one approval covers every sending mailbox), access level "Trusted" → Finish. One-time step per Workspace (~10 min to propagate). Walkthrough video: Josh's Loom "Setting Up Smartlead Gmail Mailbox Integration"
  3. Back in Smartlead, connect each of the 9–12 mailboxes via the OAuth pop-up (requires logging in as each mailbox user once — get the logins via password manager) and assign them to the BlankSlate client
OAuth gotcha: changing a connected mailbox's Google password revokes its OAuth tokens and silently disconnects it from Smartlead. Nobody rotates passwords on sending mailboxes without a heads-up so we can reconnect. (This is in the client doc too.)

Phase 5 — Warmup settings 5 min per mailbox, then wait

Enable warmup on every mailbox the day it connects:

  • Total warmup emails/day: cap around 40, starting low
  • Ramp-up increment: +4/day, or use Smartlead's auto-adjust (throttles the ramp based on each inbox's health score)
  • Reply rate: 50–60% to simulate real conversations
  • Randomize daily volume so the pattern looks human
  • Warmup never turns off — it runs alongside live campaigns permanently; teams that kill it post-launch see deliverability decay within 6–8 weeks
Gate before first cold send: 2–3 weeks of warmup minimum (30 days for brand-new domains), health scores green on every mailbox, and a spam-placement test showing inbox — not promotions/spam — on both Gmail and Outlook. When campaigns start, per-mailbox cold volume begins ~10/day and ramps toward 25–30/day, which is exactly what the SOW's 10-week ramp absorbs.
