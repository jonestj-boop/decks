Smartlead Campaign Skill — How to Build & Run Campaigns

Prospeo lead sourcing → sequence generation → Smartlead launch · For Deanna & Alex · July 2026

What This System Is

We run cold outreach through Claude Code skills in the Outreach project. Instead of clicking through Smartlead and Prospeo manually, you describe what you want in plain English and Claude runs the scripts. Three skills work together:

SkillWhat it does
campaign-strategyDesigns messaging: takes a business doc + audience, picks an angle, writes a 3–4 email sequence in Smartlead-ready format
find-leadsSources prospects via Prospeo (search → enrich with verified emails → push to Smartlead), with a local database that prevents paying twice for the same lead
smartleadManages Smartlead via API: create campaigns, save sequences, add leads, pull stats
The big idea: you don't need to memorize the commands below. Open Claude Code in the Outreach project and say things like "create a Smartlead campaign called BlankSlate — Security VP Compliance under the BlankSlate client, generate a sequence from the strategy doc, and source 50 test leads." Claude finds the skills and runs them. The commands are here so you understand what's happening and can verify it.

Before You Start — Access Checklist

  1. Claude Code installed and working on your machine
  2. Outreach project on your machine: ~/Claude Code Projects/Outreach (get a copy from Josh — includes the three skills)
  3. .env file in the Outreach project root with two keys (get values from Josh, never commit this file): 
    PROSPEO_API_KEY=...
SMARTLEAD_API_KEY=...
  4. Smartlead login — to eyeball campaigns before launch (the API stages everything; a human always reviews in the UI)
  5. uv installed — the Python runner the scripts use (curl -LsSf https://astral.sh/uv/install.sh | sh)

All commands below run from the Outreach project root. Rate limit: Smartlead allows 10 API requests per 2 seconds — the scripts handle this.

The Campaign Workflow

1 Create the campaign

uv run .claude/skills/smartlead/scripts/campaigns.py create \
  --name "BlankSlate — Security VP Compliance — Jul 2026" \
  --client-id <BLANKSLATE_CLIENT_ID>

Note the campaign ID it returns — every later step uses it. --client-id assigns the campaign to the BlankSlate client workspace in Smartlead so their data stays separated. List existing campaigns anytime with campaigns.py list.

Naming convention: {Client} — {Segment/Persona} — {Month Year}. With up to 7 BlankSlate campaigns running at once, names are how we stay sane.

2 Generate the email sequence

uv run .claude/skills/smartlead/scripts/generate_sequence.py \
  --audience "VP Compliance at private security firms" \
  --offer "Knowledge Gap Audit for compliance training" \
  --output sequence.md

Better: ask Claude to use the campaign-strategy skill with BlankSlate's strategy doc as input — it extracts value props, case studies (USAF, Mass General), and pain points, then writes the sequence. The output is a markdown file with a sending schedule and 4–5 emails using merge tags like {{first_name}} and {{company_name}}.

Human review required: read and edit sequence.md before saving. For BlankSlate, campaign copy also needs Derek's approval before launch. Never push AI copy straight to a live campaign.

3 Save the sequence to the campaign

uv run .claude/skills/smartlead/scripts/sequences.py save \
  --campaign <CAMPAIGN_ID> \
  --file sequence.md

4 Source leads (Prospeo → Smartlead)

The full pipeline searches Prospeo, enriches only verified emails, dedupes against our lead database, and pushes to the campaign:

uv run .claude/skills/find-leads/scripts/pipeline.py \
  --industry "Security & Investigations" \
  --job-title "VP Compliance,VP Operations" \
  --size "51-200" \
  --location "United States" \
  --max-per-company 2 \
  --campaign <CAMPAIGN_ID>

Or run it in two steps when you want to check list quality before spending enrichment credits:

# Search first (cheap: 1 credit per 25-result page) and review the output
uv run .claude/skills/find-leads/scripts/search_prospeo.py \
  --industry "Security & Investigations" --job-title "VP Compliance" \
  --size "51-200" --location "United States" --output results.json

# Then enrich (1 credit per verified email) and push
uv run .claude/skills/find-leads/scripts/enrich_prospeo.py \
  --input results.json --output enriched.json --verified-only
uv run .claude/skills/find-leads/scripts/smartlead_push.py \
  --input enriched.json --campaign <CAMPAIGN_ID>
Prospeo credits are the main cost. Search costs 1 credit per page (25 results); enrichment costs 1 credit per verified email. Start every new segment with a small search, check the fit of who comes back, then scale. The database at workspace/data/leads.db automatically skips people we've already enriched — never use --no-dedup unless Josh says so.

5 Review in Smartlead, then launch

In the Smartlead UI, open the campaign and verify before hitting start:

  • Sequence steps, delays, and merge tags render correctly (send yourself a test)
  • Leads loaded with correct fields — spot-check 10
  • Correct sender mailboxes attached (BlankSlate branded alt-domain mailboxes only)
  • Sending schedule respects the current ramp week's daily cap
  • Warmup has run at least 2–3 weeks on every attached mailbox and deliverability status is green

6 Monitor & iterate

Weekly, ask Claude in the Outreach project to pull campaign stats (opens, clicks, replies, positive replies by segment) and summarize what's working. This feeds the monthly optimization cycle in the SOW: kill quiet segments/angles fast, iterate copy on whatever gets replies, and generate the monthly report.

Troubleshooting

SymptomLikely cause / fix
Script errors about missing API key.env missing or not in the project root you're running from
Search returns few/zero leadsFilters too narrow — loosen one at a time (industry names must match Prospeo's list; ask Claude to check find-leads/enums)
Enrichment returns far fewer than searchedNormal — only verified emails are kept, and known leads are deduped. Check the run summary for "skipped duplicates"
Leads pushed but not visible in campaignWrong campaign ID — run campaigns.py list and confirm
API calls failing intermittentlyRate limit (10 req / 2 sec) — rerun; scripts are safe to retry
Open/reply rates tankingStop sends, check mailbox warmup health + spam placement in Smartlead before resuming. Flag Josh
