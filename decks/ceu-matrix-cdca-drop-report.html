Root Cause Analysis

4 Probable Causes — Ranked by Likelihood

Based on what we already know about the site structure, here are the most likely explanations.

Highest Probability
Cause #1

/course/ vs /product/ Canonical Conflict

CEU Matrix has duplicate URLs for every Ohio CDCA credential:

/course/ohio-cdca-preliminary-phase-i/
/product/ohio-cdca-preliminary-phase-i/
/course/ohio-cdca-phase-ii/
/product/ohio-cdca-phase-ii/

Without canonical tags resolving which URL is authoritative, Google splits ranking authority between two competing pages for the same content. This is structurally unstable — Google can flip which version it prefers at any time. When it does, the position on the "losing" URL drops sharply. If GSC is tracking one URL and Google shifted preference to the other, the data looks like a position drop when it's actually a URL preference switch.

Confirm with: URL Inspection in GSC on both versions. If Google reports different canonicals for each — or shows the non-preferred URL as a duplicate — this is the cause.
Medium Probability
Cause #2

Post-Migration Ranking Instability

The site migrated to the new build in late February 2026. Two months post-migration, rankings can still fluctuate significantly as Google re-crawls and re-evaluates the new site structure. Ohio CDCA pages may have lost whatever thin ranking signals they held on the old site and haven't rebuilt them on the new one yet. This is especially likely if the CDCA page content or URL structure changed during migration.

Confirm with: Compare current CDCA page URLs and content to what existed pre-migration. If URLs changed or content was thinned during the rebuild, that's the trigger.
Medium Probability
Cause #3

Google Algorithm Update

A confirmed Google core or spam update may have rolled out in the window of the drop. If the drop dates align with a confirmed update, the cause is algorithmic — not a site-level technical issue. The fix in that case is content quality and authority, not a technical patch.

Confirm with: Pull the exact drop start date from GSC and cross-reference against the Google Search Status Dashboard and known update calendar. If they align, this is the cause.
Lower Probability
Cause #4

Competitor Page Improvement

CASR or CE4Less may have strengthened their Ohio CDCA content, published new informational pages, or acquired a backlink that pushed them above CEU Matrix in the same window. Zero-sum ranking moves are common in competitive SERPs.

Confirm with: SEMrush position history for the same queries — if a competitor moved up at exactly the same time CEU Matrix moved down, this is a contributing factor.
Action Plan

Recommended Next Steps

In priority order — start with diagnosis before making any page changes.

  1. Pull GSC data immediately — Performance report filtered to Ohio CDCA queries, last 28 days vs. previous 28 days. Identify exact drop date and which queries are affected.
  2. Run URL Inspection on both /course/ and /product/ CDCA URLs — Confirm which version Google is treating as canonical. If Google is indexing the non-preferred URL or shows "Duplicate" status, the canonical conflict is confirmed as the primary cause.
  3. Cross-reference drop dates against Google's update calendar — If the drop date aligns with a confirmed rollout, document it and add algorithmic sensitivity as a contributing factor.
  4. Check competitor positions — Run SEMrush position history for CASR and CE4Less on the same queries. If they moved up in the same window, flag that for Everett as context.
  5. Implement canonical tags on all /course/ and /product/ duplicate pairs — Regardless of which cause is confirmed, this needs to be done. Add canonical tags pointing all four /course/ URLs to the /product/ versions (or vice versa, based on which URL holds stronger existing authority). This is a Month 2 technical deliverable and should be prioritized immediately given this signal.
  6. Submit canonical URLs for re-indexing — After canonical tags are implemented, use GSC URL Inspection to request indexing on the canonical versions. This prompts Google to re-evaluate and consolidate authority faster.
  7. Report back to Everett — Share confirmed cause + steps taken. See draft response below.
Do not make content changes to the CDCA pages until the canonical issue is resolved. Adding content to the wrong URL (the non-canonical one) will compound the problem, not fix it.
