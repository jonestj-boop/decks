Root Cause — Data Confirmed

What's Causing the Drop

One primary cause, confirmed in live data and pre-identified in the Month 1 audit. Two secondary factors worth noting.

Confirmed — Live Data
Root Cause

/course/ vs. /product/ Canonical Conflict — Google Switching Preference

Without canonical tags, Google decides on its own which URL to rank for each duplicate pair — and it can change that decision. The GA4 data shows it already completed this switch for Phase II:

URL PairRanking URLNon-Ranking URLStatus
Phase I — Preliminary /product/ (107 sessions) /course/ (19 sessions) Split — unstable
Phase II /product/ (13 sessions, declining) /course/ (0 sessions) Switch complete

When Google makes this switch, the URL that was ranking before goes to zero — which shows up as a sudden position drop in GSC average position. The queries are still being searched; Google just stopped ranking the previously-preferred URL. Neither URL benefits, because the ranking authority that was split between them is now fully on one — but that one hasn't been reinforced with the content and links needed to hold strong positions.

📋

Documented in Month 1 audit — scheduled for Month 2

Tech Implementation Item #5: "Canonical tags: /course/ vs /product/ credential pairs (4 pairs)" — This fix was already on the plan. The drop is happening because the fix hasn't been implemented yet. We're accelerating it to this week.

Secondary Factor
Contributing

Post-Migration Crawl Re-evaluation

The site migrated to the new build in late February 2026. Two months post-migration, Google is still re-evaluating the new site architecture. The canonical ambiguity that was tolerated on the old site may have triggered a harder re-evaluation on the new one. The fact that Phase II completed the switch (to zero /course/ traffic) while Phase I is still mid-transition suggests this re-evaluation is ongoing.

Monitor
Secondary

Google Core Update Window

Google ran a confirmed update in the same window as the reported drop. Core updates often force Google to resolve previously ambiguous signals — like two URLs competing for the same keyword. If the update window aligns with the Phase II drop date, the update likely accelerated the canonical preference switch rather than causing a new problem. The fix is identical either way.

Resolution Plan

What We're Doing — and When

Sequenced by priority. Steps 1 and 2 are happening this week.

  1. Add canonical tags to all four /course/ vs /product/ duplicate pairs This Week
    For each credential pair, add <link rel="canonical"> to the /course/ version pointing to the /product/ version (GA4 confirms /product/ is Google's preferred URL for both Phase I and Phase II). This tells Google definitively which URL to rank and stops the switching behavior.
    /course/ohio-cdca-preliminary-phase-i/ → canonical: /product/ohio-cdca-preliminary-phase-i/
    /course/ohio-cdca-phase-ii/ → canonical: /product/ohio-cdca-phase-ii/
  2. Submit canonical /product/ URLs for re-indexing in GSC This Week
    After canonical tags are live, request indexing on all four /product/ CDCA URLs via GSC URL Inspection. This prompts Google to re-crawl and consolidate ranking authority faster rather than waiting for the next natural crawl cycle (which could be days to weeks).
  3. Add service account to GSC property for query-level data This Week
    We have GA4 access confirmed. For GSC query-level data (exact positions per keyword), the CEU Matrix GSC property needs to grant access to: embertribe-content-tools@embertribe-content-tools.iam.gserviceaccount.com. This lets us pull the exact position-by-position drop per query and track recovery automatically.
  4. Monitor the /product/ CDCA pages weekly in GA4 Ongoing
    Track organic sessions to /product/ohio-cdca-preliminary-phase-i and /product/ohio-cdca-phase-ii weekly. After canonical tags are implemented, expect /product/ Phase I to hold or grow from 107/week. Phase II recovery will take 2–4 weeks as Google consolidates the signal.
  5. Fix the broken internal link from the CDCA blog post This Week
    The post /cdca-certification-in-ohio-requirements-phase-1-training-and-your-next-steps has a broken link to /courses-by-state/ohio (404, confirmed in site audit). This post is getting new organic traffic (5 sessions last 7d vs 1 prior) and is a bottom-of-funnel entry point for buyers. The broken link is losing potential purchases.
  6. Build the Ohio unified hub — Month 2 priority Month 2
    Once canonicals are resolved, the /product/ CDCA pages will have consolidated authority but still lack the informational content depth needed to rank for credential-research queries ("ohio cdca requirements," "cdca certification ohio"). The Ohio hub at /ohio-substance-abuse-certification/ becomes the authority anchor for all CDCA organic traffic. This is the Month 2 cornerstone build.
Do not add content to the /course/ CDCA URLs. Once canonical tags point /course/ → /product/, the /course/ URLs will no longer be indexed. Any content on them will be invisible to Google. All content investment should go on the canonical /product/ pages and the new Ohio hub.
