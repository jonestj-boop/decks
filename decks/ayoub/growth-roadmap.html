Technical Roadblocks

Mobile LCP of 15.6 seconds. Google won't rank a page it can't load fast on a phone.

Every content and local effort is taxed by site speed. A 15.6-second load time on mobile means customers leave before they even see the service. Google sees the same signal and depresses rankings accordingly. This is the highest-leverage technical fix in the roadmap.

PageSpeed Insights — key pages

Page Mobile Desktop Top Issue

/services/carpet-cleaning/ timeout 56 TBT 4,360ms desktop — JS blocking
/locations/rockville/ 41 65 LCP 9.5s mobile / unused CSS 310ms
P1: Homepage mobile LCP = 15.6 seconds
Google's Core Web Vital threshold for "good" is under 2.5 seconds. At 15.6s, the homepage fails the LCP benchmark by 6x. This single fix — likely a hero image that isn't lazy-loaded or an above-fold render-blocking script — would have the biggest ranking impact of any change on the site.

Fix priority

P1 Homepage mobile LCP

Identify and lazy-load hero image. Remove or defer above-fold render-blocking scripts. Expected gain: 8–12 ranking positions on homepage-indexed queries.

P2 Unused CSS / JS (homepage)

750ms from unused CSS, 290ms from unminified JS. Combined fix recovers over 1 second of load time — meaningful for mobile rankings.

P3 Rockville location page LCP 9.5s

The most important local landing page is also slow on mobile. Fix before optimizing Kensington/Potomac and building additional location pages — don't build on a broken foundation.

Platform note: WordPress + Elementor is the common culprit here. Elementor generates unused CSS for every widget on every page. A caching plugin (WP Rocket) + CSS optimization pass typically resolves most of this without a full rebuild.
